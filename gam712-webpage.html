Theme: The illusion of choice

Project Overview

This project involved creating a narrative-driven game titled "StarStory" which explores the concept of the illusion of choice. Players navigate a tense scenario aboard a starship invaded by aliens, making decisions that appear to influence the story while ultimately leading to a predetermined outcome.

Development Process

I started by thinking about how Telltale Games approach their narrative games. Normally, the player can't actually change the outcome but can make smaller in-story decisions. I don't usually create narrative games, so this type of project was a challenge for me. Everyone else was working with Twine, but I wanted to stick with learning Unity. I looked into narrative plugins and found Yarn Spinner. It was amazingly easy to use and perfect for someone technical, as you can call methods and link variables in their Yarn script or in your own script.

From there, I started to develop a narrative story. I really like Star Trek, so I wanted to create a plotline that could have happened in the show. The general idea was that aliens have boarded the ship. In the first run of the game, most players would try diplomacy. From that, they would be given secret variables that change how the game plays out, but not the ending itself.

In the next run, players have more information and can quickly figure out what they need to do to end the game. The game goes in an unending loop until the player figures it out.

Key Features

  • Character relationship mechanics that influence game outcomes
  • Dynamic dialogue options based on previous player choices

Technical Implementation

The narrative system was built using a prebuilt dialogue manager that tracks state changes and triggers appropriate story branches. Player choices are stored in a persistent data structure, allowing for complex decision trees and callback references to earlier choices.

Theme: large scale small scope

Project Overview

This project required me to create a game that had a large scale but was small in scope.

Development Process

This project required me to create a game that had a large scale but was small in scope. I interpreted this as making a game with a small setting that has the potential to feel larger. My setting was a blacksmith who forges swords. The sword's creation is based on mini-games such as forging, quenching, sharpening, and enchanting, which determine its stats and influence a kingdom-level combat system.

During development, I focused on creating engaging mini-games that felt rewarding and intuitive. Each mini-game was designed to represent a specific aspect of the blacksmithing process, ensuring variety and depth. While I initially planned to implement a full kingdom combat system, time constraints led me to create a simplified sword vs. sword display to showcase the impact of the player's choices.

Additionally, I utilized Yarn Spinner to provide interactive tutorials and guidance for players, building on my experience from the previous project. This allowed me to create a more immersive and user-friendly experience while maintaining a consistent narrative tone.

Key Features

  • Multi mini-games
  • influences something larger than the blacksmith itself

Technical Implementation

Mini games built with unity UI.

Theme: metaphor

Project Overview

I had to create a game based on the theme of metaphor, I decided to take that quite literally and use the metaphor of standing on the shoulders of giants.

Development Process

I initially struggled to come up with a solid idea for this project. I experimented with a penguin puzzle game (though I honestly had no clear direction for it), then moved on to creating a physics-based game, but nothing seemed to click. I even tried developing a simple platformer, but it still felt uninspired.

Eventually, I had a breakthrough: I decided to use the player's previous platform runs as a progression mechanic. This concept gave the platformer a unique twist and a sense of purpose. I themed the game around a hellish landscape, which added a fitting atmosphere to the mechanics. While the graphics were quite basic due to time constraints, the gameplay felt engaging and had the potential to evolve into a larger, more polished game in the future. The metaphor was that in academics, we use research others have already done for us, so I was thinking that the player would be standing on the shoulders of giants that have already started to lay out the path for them, allowing them to escape.

Key Features

  • Player progression mechanics based on previous platform runs
  • Physics-based interactions that enhance platforming challenges
  • Dynamic level design influenced by player actions

Technical Implementation

The progression mechanic was implemented by recording player movements and replaying them as interactive elements in subsequent runs. Custom physics systems were developed to ensure smooth interactions between the player and the environment. Additionally, procedural level adjustments were introduced to adapt to the player's progression and maintain engagement.
